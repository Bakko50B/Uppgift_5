Diagram

På sidan diagram finns min lösning på denna uppgift. Jag har använt chart.js för att skapa de olika diagramen som uppgiften ska innehålle. För bättre styling av tabellen och dess data har jag använt chartjs-plugin-datalabels.

Jag har installerat chart js och chartjs-plugin-datalabels som mpn moduler i min Parceldrivna webbplats.

Karta

På sidan karta finns en karta med sök­möjligheter. Utöver att söka på plats finns det även en positions­möjlighet där platsen man är på visas. För att använda denna funktion krävs att användaren tillåter GPS-positionering via webbläsarens gränssnitt.
För att se sin positon trycker man på knappen "GPS"

I samband med att jag kände mig redo att lämna in så studerade jag dokumentationen till Nominatum ytterligare. Då insåg jag att det borde kunna vara möjligt att få positionen genom att klicka på kartan, och på det sättet få en adress där klicket skedde.
Det gick bra.

Nästa idé jag fick handlade om att använda adressen som genereras från klicket och skicka in till en söktjänst för att kunna visa en länk med adressen som sökkriterie. Om man skulle hamna på ett intressant ställe när man klickar kan det vara kul att läsa mer om platsen. För att omvandla plats­informationen till att säkert fungera med URLer använde jag "encodeURIComponent". Länken placerades i popupelementet som ett tillägg till adressinformationen.

Det känns kul att kunna få idéer och sedan genomföra dessa med att leta i manualer.

Tekniken jag använt på sidan karta (map.html) är Nominatim, OpenStreetMap och Leaflet. Valet av denna teknik är vald utifrån de olika karttjänsternas olika avtal. För mig passas valet av OpenStreetMap bäst.

Erfarenheter - slutsats

Att få jobba på egen hand med dessa uppgifter har varit kul. En del saker flöt på väldigt bra, medan annat tog längre tid. Att visa diagram på den statistik som skulle inhämtas flöt på relativt lätt.

Diagram

De inbyggda jsfunktionerna som sort och filter kom väl till pass.

Att producera både stapeldiagram och cirkeldiagram gick ganska bra. Dock behövdes det lite css-kod för att få stapeldiagrammet "lagom" högt.

En svårare uppgift att lösa var att placera texten som beskriver vad staplarna representerar.
Som standard visas den texten under staplarna med en lutning på några grader. För att lättare kunna avgöra vad som staplarna representerar ville jag placera texten inne i staplarna. För att lösa den delen fick jag läsa dokumentation, samt använda ett plugin till chart js som hanterar layouten.

Karta

Kartan och sökning på ort var kul att göra. Det flöt på väldigt bra. Efter en föreläsning där GPS togs upp som ett ämne lade jag till möjligheten att visa aktuell position. Jag hade redan tidigare lyckats lösa baskravet på den delen i uppgiften, så det passade bra att utöka "min karta" med att visa position. Den dokumentation som jag hittade angående de delar som min karta består av upplevde jag som bra och enkel att följa. De exempel som jag hittade var också till stor hjälp.

En sista funktion implementerade jag genom att hämta longitud och latitud på den plats som användaren pekar på, för att sedan skicka de hittade koordinaterna till Nominatim som svarar med platsens namn om det finns ett namn för platsen. Extra extra blev länken för sökning.
En kul extra (extra) funktion.
